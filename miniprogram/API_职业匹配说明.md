# 职业匹配API说明

## 接口地址
`POST /occupation/match`

## 重要更新
**AI生成职业时，必须为每个职业匹配一个合适的性格类型代码**

## 返回数据格式

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": "occupation_001",
      "name": "临床试验专员",
      "majors": ["药学", "临床医学", "生物"],
      "skills": ["数据分析", "临床研究", "GCP规范"],
      "personalityCode": "ISC",
      "industryHeat": "B",
      "reason": "新药研发产业链关键环节，行业规范提升专业价值，出差多适合喜欢走动者",
      "description": "临床试验专员负责协调和监督临床试验的实施过程...",
      "careerPath": "初级CRA → 高级CRA → 项目经理..."
    }
  ]
}
```

## 性格类型代码说明

AI生成职业时，应根据职业特点选择合适的性格类型代码：

### 常用性格代码及适用职业

| 代码 | 性格特点 | 适合职业示例 |
|------|---------|-------------|
| **ISC型** | 内向、善于分析、注重细节、关心他人 | 临床医生、护士、医学检验、心理咨询 |
| **IRC型** | 内向、实际且理性、逻辑严密 | 软件工程师、数据分析师、质量工程师 |
| **IEC型** | 内向、有研究精神、具备管理能力 | 技术经理、产品经理、研发主管 |
| **ESC型** | 外向、善于组织协调、责任心强 | 项目经理、人力资源、行政管理 |
| **SAE型** | 外向、富有创意和领导力 | 品牌经理、公关总监、市场营销 |
| **AIC型** | 内向、具有创造力和研究精神 | UI设计师、产品设计、用户研究 |
| **CRI型** | 理性严密、善于分析、注重细节 | 数据科学家、算法工程师、财务分析 |

### 选择建议

1. **医疗护理类**：ISC、ISR、ISA
2. **技术工程类**：IRC、IEC、CRI
3. **管理类**：ESC、ERC、IEC
4. **设计艺术类**：AIC、AES、SAI
5. **销售市场类**：SAE、SRA、SEC

## 示例

### 医疗类职业
```json
{
  "name": "肿瘤内科医师",
  "personalityCode": "ISC",
  "reason": "需要细致的诊断能力和对患者的同理心"
}
```

### 技术类职业
```json
{
  "name": "后端开发工程师",
  "personalityCode": "IRC",
  "reason": "需要逻辑严密、善于解决复杂技术问题"
}
```

### 管理类职业
```json
{
  "name": "项目经理",
  "personalityCode": "ESC",
  "reason": "需要组织协调能力和责任心"
}
```

## 重要提示
- ❌ 不要使用"通用"或空值
- ✅ 必须为每个职业指定具体的性格代码
- ✅ 性格代码应与职业特点匹配
- ✅ 前端会显示为"XX型"格式（如"ISC型"）
