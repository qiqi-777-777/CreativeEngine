# 创享引擎（CreativeEngine）项目详细介绍

## 一、项目概述

**创享引擎（CreativeEngine）** 是一个面向青年创业者、大学生创新创业团队以及早期项目负责人的一站式创业赋能平台。项目以微信小程序作为主要用户入口，配套 Spring Boot 后端服务，围绕“方向判断、政策获取、资源连接、AI 辅助决策”四个核心场景，帮助用户从创业想法萌芽走向项目规划、政策申报与资源对接。

平台不是单纯的信息展示工具，而是一个融合 **创业服务、政策匹配、AI 问答、职业与行业分析、商业计划书辅助生成** 的综合型应用。用户可以在小程序中完成政策查询、职业匹配、行业图谱查看、BP 智写、专家对接、AI 咨询等操作，后端则负责数据接口、业务处理、AI 能力接入与用户会话管理。

## 二、项目定位

创享引擎的目标用户主要包括：

- 有创业想法但缺少方向判断的大学生与青年创业者
- 正在准备创新创业比赛、商业计划书或项目路演的团队
- 需要了解创业政策、补贴条件、申报流程的初创项目
- 希望对接专家、资源、场地、政策与服务机构的创业群体
- 希望借助 AI 提高资料整理、问题咨询和方案生成效率的用户

项目的核心价值可以概括为：

- **降低创业信息门槛**：把分散的政策、行业、资源信息集中到一个移动端入口。
- **提升决策效率**：通过职业匹配、政策匹配、AI 问答等能力，帮助用户快速获取建议。
- **强化创业执行能力**：通过 BP 智写、专家对接、我的匹配等模块，把想法转化为可执行材料。
- **形成个人创业工作台**：在“我的”页面沉淀匹配结果、常用服务和个人数据。

## 三、整体架构

项目采用前后端分离架构：

- **前端**：微信小程序原生开发，使用 WXML、WXSS、JavaScript 实现页面与交互。
- **后端**：Spring Boot 3.1.5，提供 RESTful API。
- **数据库**：MySQL，存储政策、用户、会话、轮播图、职业匹配等业务数据。
- **AI 接入**：后端通过 OkHttp 调用 DeepSeek API，实现 AI 助手对话与创业咨询能力。
- **数据访问层**：MyBatis Plus 负责实体映射、查询与业务数据操作。

整体数据流如下：

```text
微信小程序页面
  ↓
utils/request.js 请求封装
  ↓
Spring Boot REST API
  ↓
Service 业务逻辑层
  ↓
Mapper / MyBatis Plus
  ↓
MySQL 数据库

AI 相关请求：
微信小程序 AI 页面
  ↓
/api/ai/chat
  ↓
AiAssistantController
  ↓
DeepSeek API
  ↓
返回智能问答结果
```

## 四、前端小程序模块

### 1. 首页（index）

首页是整个小程序的核心入口，承担功能导航和服务聚合的作用。页面通过底部 TabBar 与“AI助手”“我的”形成主导航结构。

首页主要功能包括：

- 展示项目品牌与欢迎信息
- 加载首页轮播图数据
- 提供核心服务入口
- 跳转到政策匹配、职业匹配、图说解读、资源对接等功能页面

首页强调快速入口和高频服务聚合，适合用户第一次进入小程序时快速了解平台能做什么。

### 2. AI 助手（ai-assistant）

AI 助手是平台的智能问答模块，后端通过 `/api/ai/session/create` 创建会话，通过 `/api/ai/chat` 进行对话交互。

主要能力包括：

- 创业政策咨询
- 商业计划书写作建议
- 企业注册流程说明
- 创业补贴与申报材料指导
- 项目方向分析
- 通用创业问题答疑

AI 助手采用会话式体验，用户可以连续提问，系统根据上下文返回更贴近创业场景的回答。

### 3. 我的（mine）

“我的”页面是用户个人工作台，聚合用户数据、常用服务和个人管理入口。

主要内容包括：

- 用户登录 / 注册入口
- 数据概览：已读行业、适配政策、对接资源等统计
- 常用服务：行业图谱、我的匹配、BP 智写、专家对接
- 菜单管理：我的申请、企业档案、资质认证、消息通知、帮助与客服、设置

该页面的定位不是单纯个人资料页，而是创业过程中的个人控制台。

### 4. 政策匹配与政策详情（policy、policy-detail）

政策模块用于展示和查询创业扶持政策。用户可以浏览政策列表，进入详情页查看具体申报条件、适用对象和政策说明。

当前政策匹配已经接入中国政府网公开政策数据增强能力。项目提供 `scripts/import-gov-graduate-policies.js` 抓取脚本，用于按关键词低频抓取 `gov.cn` 公开政策页面，并生成可导入 MySQL 的 SQL 文件。当前默认政策关键词包括：

- 毕业生
- 创业
- 大学生

搜索时既支持单独关键词，也支持组合输入。例如：

- `毕业生`
- `创业`
- `大学生`
- `大学生创业`
- `大学生 创业`
- `毕业生+创业`
- `毕业生,创业,大学生`

后端会把组合输入拆成多个条件，并按“同时满足这些条件”的方式进行匹配。前端输入框的即时筛选也使用同样的拆词逻辑，保证列表筛选和点击 MATCH 后的后端查询结果一致。

政策详情页会将正文与来源信息分开展示。抓取脚本和前端工具会清理中国政府网页面中的导航、打印、责任编辑、站点链接等杂项，让详情页优先展示政策正文段落；来源和原文链接则作为补充信息放在底部。

后端对应接口主要包括：

- `/api/policy-data/list`
- `/api/policy-data/detail/{id}`
- `/api/policy-data/ask`

该模块解决政策信息分散、用户难以理解和筛选的问题。

### 5. 职业匹配与匹配结果（career-match、career-match-result）

职业匹配模块用于帮助用户根据个人背景、专业、技能、兴趣等信息获得更适合的发展方向或创业切入点。

后端对应接口包括：

- `/api/occupation/majors`
- `/api/occupation/skills`
- `/api/occupation/personalities`
- `/api/occupation/match`

匹配结果页用于展示推荐方向、推荐理由和后续行动建议。

### 6. 图说解读（image-interpretation）

图说解读模块面向图片、图表、报告截图等复杂信息场景。用户可以借助该页面对图像内容进行解释和分析，适用于阅读行业图表、政策截图、商业模式图、调研材料等。

该模块的价值在于降低图表理解门槛，让用户更快从图片信息中提取关键结论。

### 7. 行业图谱（industry-atlas）

行业图谱模块用于呈现行业结构、产业链关系、发展方向与机会点。它帮助创业者从宏观角度理解行业上下游、竞争格局和潜在切入位置。

适用场景包括：

- 创业方向调研
- 行业竞品分析
- 项目路演材料准备
- 商业计划书市场分析部分辅助

### 8. 我的匹配（my-match）

我的匹配用于集中管理用户已经生成或保存的匹配结果，例如职业匹配、政策匹配等。它让用户不必每次重新填写信息，可以持续查看历史结果和后续建议。

### 9. BP 智写（bp-writer）

BP 智写是商业计划书辅助生成模块，面向创新创业比赛、融资路演和项目申报材料准备场景。

模块目标包括：

- 引导用户梳理项目定位
- 辅助生成商业计划书结构
- 提供项目亮点、市场痛点、商业模式等写作参考
- 降低非专业用户撰写 BP 的难度

当前实现中，BP 智写会先创建 AI 会话，再调用 `/api/ai/chat` 生成商业计划书内容；当 AI 接口不可用时，页面提供本地降级模板，保证用户仍能获得一份基础 BP 草稿。

### 10. 资源对接（resource）

资源对接页是项目中已有的独立页面，当前更偏向“校园工作 / 资源机会列表”场景。页面内置多类机会数据，支持搜索和分类筛选。

当前分类包括：

- 全部
- 基础便捷
- 学术辅助
- 技能变现
- 灵活服务

该页面主要用于展示兼职、技能服务、校园资源等机会信息，可以作为后续“创业资源对接”模块的基础版本。后续如果接入后端，可以扩展为资源发布、报名申请、预约联系和状态追踪。

### 11. 专家对接（expert-connect）

专家对接模块用于连接创业服务资源，例如行业专家、财务顾问、法律顾问、创业导师等。

主要解决的问题是：早期创业者缺少外部专业支持，不知道该向谁咨询、如何获取指导。

当前页面以本地专家数据为主，包含专家列表、分类筛选、详情弹窗、服务方式展示等交互，适合后续对接真实专家库与预约接口。

### 12. 登录与注册（login、register）

登录注册模块负责用户身份入口，支持后续用户信息、个人中心、历史记录和业务数据沉淀。

后端认证接口主要包括：

- `/api/auth/login`
- `/api/auth/register`
- `/api/auth/logout`
- `/api/auth/userInfo`

前端登录页还预留了微信登录交互入口；当前后端已实现账号密码登录注册相关接口，微信登录接口仍属于后续可扩展项。

## 五、后端服务模块

### 1. 基础服务

后端基于 Spring Boot 构建，默认端口为 `8080`。项目提供基础健康检查与测试接口：

- `/`
- `/health`
- `/api/test`

这些接口用于快速确认后端服务是否正常启动。

### 2. Banner 管理

`BannerController` 负责首页轮播图相关接口：

- `/api/banner/list`：小程序首页获取轮播图
- `/api/banner/admin/list`：管理端轮播图列表
- `/api/banner/admin/upload`：上传轮播图
- `/api/banner/admin/update`：更新轮播图
- `/api/banner/admin/delete/{id}`：删除轮播图

该模块支撑首页内容运营。

### 3. AI 助手服务

`AiAssistantController` 是 AI 能力的主要入口：

- `/api/ai/session/create`：创建 AI 会话
- `/api/ai/chat`：发送用户问题并获取 AI 回复
- `/api/ai/history`：查询历史消息
- `/api/ai/sessions`：查询会话列表

后端通过 DeepSeek API 完成大模型调用，并结合创业场景设定系统提示词，使 AI 回答更聚焦于创业政策、商业计划、注册流程、资金申报等主题。

### 4. 职业匹配服务

`OccupationController` 提供职业匹配相关数据与计算入口：

- `/api/occupation/majors`
- `/api/occupation/skills`
- `/api/occupation/personalities`
- `/api/occupation/match`

该模块为职业匹配页面提供选项数据与匹配结果。当前匹配逻辑会先基于数据库中的职业数据计算专业、技能、性格匹配分数；如果数据库匹配结果不足，会调用 DeepSeek 生成补充推荐。

### 5. 政策数据服务

`PolicyDataController` 提供政策列表、详情和问答能力：

- `/api/policy-data/list`
- `/api/policy-data/detail/{id}`
- `/api/policy-data/ask`

政策模块是平台帮助创业者理解扶持政策、补贴条件与申报方向的重要能力。当前 `/api/policy-data/list` 支持 `keyword` 查询参数，并支持多关键词拆分搜索。`PolicyDataServiceImpl` 会识别“高校毕业生、毕业生、大学生、创业”等常用政策关键词，能够处理 `大学生创业` 这类连续输入，也能处理空格、逗号、顿号、加号等分隔方式。

### 6. 用户认证服务

`LoginController` 提供用户认证相关能力：

- `/api/auth/login`
- `/api/auth/register`
- `/api/auth/logout`
- `/api/auth/userInfo`

它支撑用户登录注册、身份状态维护和个人信息查询。

### 7. 通用配置与静态资源

项目后端还包含以下通用能力：

- `CorsConfig`：允许跨域请求，便于小程序和本地调试访问后端。
- `WebConfig`：将 `/uploads/**` 映射到项目根目录下的 `uploads` 文件夹，用于访问上传后的轮播图等静态资源。
- `GlobalExceptionHandler`：统一处理业务异常和系统异常。
- `Result` / `ResultCode`：提供统一响应结构。

## 九、运行方式

### 1. 后端运行

环境要求：

- JDK 17
- Maven 3.6+
- MySQL 8.0+

数据库准备：

```sql
CREATE DATABASE creative_engine DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

导入项目中的数据库脚本：

```text
creative_engine.sql
```

配置环境变量或本地配置：

```text
DB_PASSWORD=你的数据库密码
DEEPSEEK_API_KEY=你的 DeepSeek API Key
```

真实 DeepSeek API Key 不应提交到 Git。项目的 `application.yml` 只读取 `${DEEPSEEK_API_KEY:}` 占位变量；如果使用 `src/main/resources/application-secret.yml` 保存本地密钥，该文件已经被 `.gitignore` 忽略。

导入政策数据：

```bash
node scripts/import-gov-graduate-policies.js "--keyword=毕业生,创业,大学生" --pages=1 --output=scripts/generated/graduate_policies.sql
mysql -uroot -p creative_engine < scripts/generated/graduate_policies.sql
```

生成的 SQL 会按政策标题去重：新政策会插入，已存在政策会刷新关键词和正文内容，避免重复堆积。

启动后端：

```bash
mvn spring-boot:run
```

## 十、核心业务流程

### 1. AI 咨询流程

```text
用户进入 AI 助手
  ↓
前端创建会话
  ↓
用户输入创业问题
  ↓
后端接收问题并组织上下文
  ↓
调用 DeepSeek API
  ↓
返回 AI 回复
  ↓
前端展示对话内容
```

### 2. 职业匹配流程

```text
用户进入职业匹配页面
  ↓
选择专业、技能、性格等信息
  ↓
提交匹配请求
  ↓
后端根据规则和数据生成匹配结果
  ↓
前端展示推荐方向与说明
```

### 3. 政策查询流程

```text
用户进入政策模块
  ↓
输入单个关键词或组合关键词
  ↓
前后端统一拆分关键词
  ↓
后端查询政策数据
  ↓
返回政策列表、详情或问答结果
  ↓
用户查看申报条件和政策说明
```

## 十一、项目特色

### 1. 面向创业场景，而不是通用工具

项目中的 AI、政策、职业、资源、BP 等模块都围绕创业者真实场景设计，不是孤立功能堆叠。

### 2. 移动端入口轻量高效

微信小程序降低了用户使用门槛，适合在校园、孵化器、比赛现场、政策宣讲等场景快速传播和使用。

### 3. 前后端结构清晰

前端负责页面体验和交互，后端负责数据接口、AI 调用和业务逻辑，便于后续扩展管理端、Web 端或更多数据服务。

### 4. AI 能力可持续扩展

当前已接入 DeepSeek 对话能力，未来可以继续扩展到：

- BP 自动生成
- 政策智能匹配
- 图像理解
- 项目诊断报告
- 路演问答模拟

### 5. 具备完整创业服务闭环

项目覆盖“找方向、查政策、写材料、问 AI、连资源、看结果”的连续流程，具备从信息服务发展为创业工作台的潜力。

## 十三、总结

创享引擎是一个以青年创业者为中心的智能创业服务平台。它通过微信小程序提供轻量、便捷的使用入口，通过 Spring Boot 后端提供稳定的数据与 AI 服务，通过政策、职业、行业、资源、BP 等模块帮助用户完成从想法到行动的关键步骤。

项目当前已经具备较完整的业务雏形和技术结构，后续可以继续围绕数据沉淀、AI 深度赋能、创业服务闭环和管理端建设进行增强，逐步发展为面向校园创业、青年创业和地方创新服务的综合平台。
