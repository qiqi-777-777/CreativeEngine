# 创享引擎（CreativeEngine）项目详细介绍

## 一、项目概述

创享引擎（CreativeEngine）是一套面向大学生、青年创业者、创新创业比赛团队和早期项目负责人的一站式创业赋能平台。项目以微信小程序作为主要使用入口，以 Spring Boot 后端作为数据与 AI 能力支撑，围绕“找方向、查政策、问 AI、写材料、连资源、管成果”的创业链路，帮助用户把分散的信息和模糊的想法转化为可理解、可整理、可持续推进的行动方案。

项目并不是单一的信息展示工具，而是一个结合政策查询、职业匹配、行业分析、AI 咨询、商业计划书辅助生成、资源对接和个人工作台的综合型应用。用户可以在小程序中完成政策检索、政策收藏、职业匹配、BP 草稿生成、AI 问答、通知查看、个人资料维护等操作；后端则负责接口服务、业务处理、数据库读写、DeepSeek API 调用和用户数据沉淀。

## 二、项目定位与目标用户

### 1. 项目定位

创享引擎定位为“青年创业服务入口”和“创业过程辅助工作台”。它重点解决早期创业用户常见的三个问题：

- 不知道适合从哪个方向切入。
- 不知道可以申请哪些政策、补贴或服务资源。
- 缺少将想法整理成材料、方案和行动路径的工具。

因此，项目将政策、职业、行业、AI、资源和个人记录整合到同一个移动端入口中，让用户可以从一次咨询、一次匹配或一次政策查询开始，逐步沉淀自己的创业信息资产。

### 2. 目标用户

- 有创业想法但缺少方向判断的大学生和青年创业者。
- 正在准备创新创业比赛、路演、项目申报或商业计划书的团队。
- 需要了解创业政策、毕业生政策、大学生就业创业扶持政策的用户。
- 希望对接专家、场地、服务机构、校园机会或创业资源的早期项目。
- 希望借助 AI 提高资料整理、问题咨询和方案生成效率的用户。

## 三、核心价值

- **高品质的视觉交互**：全面重构前端 UI，采用极简高端杂志风格排版与 3D 黏土风（Claymorphism），搭配错落式滑入动画与流畅物理反馈，为青年创客打造高级、现代的产品体验。
- **降低创业信息门槛**：将分散在政策网站、创业服务机构和行业资料中的信息聚合到小程序入口。
- **提高决策效率**：通过职业匹配、政策匹配和 AI 问答，帮助用户更快形成判断。
- **辅助材料生成**：通过 BP 智写和导出能力，将创业想法整理成更规范的文本材料。
- **形成个人工作台**：沉淀政策收藏、职业记录、BP 草稿、通知和个人资料，方便持续跟进。
- **具备扩展基础**：前后端分离、模块清晰，后续可扩展管理后台、资源预约、微信登录和更多 AI 场景。

## 四、整体架构

项目采用前后端分离架构：

- **小程序端**：微信小程序原生开发，使用 WXML、WXSS 和 JavaScript 实现页面、交互与接口调用。
- **后端服务**：Spring Boot 3.1.5，提供 REST API。
- **数据库**：MySQL，存储用户、政策、职业、Banner、收藏、草稿、通知等业务数据。
- **AI 接入**：后端通过 OkHttp 调用 DeepSeek API，向小程序提供创业问答和内容生成能力。
- **数据访问层**：MyBatis Plus 负责实体映射和数据库操作。
- **脚本层**：Node.js 脚本负责公开政策数据抓取、SQL 生成和定时导入。

数据流如下：

```text
微信小程序页面
  -> miniprogram/utils/request.js
  -> Spring Boot REST API
  -> Controller
  -> Service / Mapper
  -> MySQL

AI 相关请求：
微信小程序页面
  -> /api/ai/session/create 或 /api/ai/chat
  -> AiAssistantController
  -> DeepSeekService
  -> DeepSeek API
  -> 返回 AI 回复
```

## 五、功能模块介绍

### 1. 首页

首页是平台的核心入口，采用高级极简杂志风格设计，负责展示品牌信息、轮播图和高频服务入口。用户可以从首页快速进入政策匹配、职业匹配、图说解读、资源对接等页面。

对应页面：

- `miniprogram/pages/index`

后端能力：

- `/api/banner/list`：获取首页轮播图。

### 2. AI 助手

AI 助手是平台的智能问答模块，面向创业咨询、政策理解、项目分析和材料写作等场景。后端通过 DeepSeek API 完成大模型调用，并提供会话创建、聊天、历史记录和会话列表接口。

主要能力：

- 创业问题咨询。
- 政策理解和申请建议。
- 商业计划书写作建议。
- 企业注册、补贴申请、项目路演等流程答疑。
- 根据上下文进行连续对话。

对应页面：

- `miniprogram/pages/ai-assistant`

核心接口：

- `POST /api/ai/session/create`
- `POST /api/ai/chat`
- `GET /api/ai/history`
- `GET /api/ai/sessions`

### 3. 政策匹配与政策详情

政策模块用于展示、查询和理解创业相关政策。页面引入了极简黏土风（Claymorphism）设计，打造高级质感。用户可以通过关键词检索政策列表，查看政策详情，并在详情页围绕政策内容发起 AI 问答。

项目提供了 `scripts/import-gov-graduate-policies.js`，可按关键词从 `gov.cn` 公开页面抓取政策内容，并生成可导入 MySQL 的 SQL 文件。脚本支持按标题去重：新政策会插入，已存在政策会更新关键词和正文内容。

支持的使用场景：

- 查询“毕业生”“创业”“大学生”“就业”等相关政策。
- 查看政策适用对象、政策说明和正文内容。
- 对政策内容进行 AI 问答。
- 收藏重要政策，后续在个人中心查看。

对应页面：

- `miniprogram/pages/policy`
- `miniprogram/pages/policy-detail`
- `miniprogram/pages/policy-favorites`

核心接口：

- `GET /api/policy-data/list`
- `GET /api/policy-data/detail/{id}`
- `POST /api/policy-data/ask`
- `GET /api/user-data/policy-favorites`
- `POST /api/user-data/policy-favorites`
- `DELETE /api/user-data/policy-favorites`

### 4. 职业匹配

职业匹配模块帮助用户基于专业、技能、性格等信息获得适合的发展方向或创业切入点。该模块 UI 结合了黏土风与错落入场动效，大幅提升交互趣味性。后端会根据数据库中的职业数据计算匹配结果；当数据库结果不足时，可以结合 DeepSeek 生成补充建议。

对应页面：

- `miniprogram/pages/career-match`
- `miniprogram/pages/career-match-result`
- `miniprogram/pages/career-records`

核心接口：

- `GET /api/occupation/majors`
- `GET /api/occupation/skills`
- `GET /api/occupation/personalities`
- `POST /api/occupation/match`
- `GET /api/user-data/career-records`
- `POST /api/user-data/career-records`
- `DELETE /api/user-data/career-records/{recordKey}`

### 5. BP 智写与草稿管理

BP 智写面向创新创业比赛、路演、融资沟通和项目申报材料准备。用户填写项目基础信息后，系统可调用 AI 生成商业计划书内容；当 AI 接口不可用时，页面也保留本地模板兜底，确保用户仍可获得基础草稿。

BP 草稿可保存到后端，方便用户后续继续编辑。后端还提供 Word 和 PDF 导出接口。

对应页面：

- `miniprogram/pages/bp-writer`
- `miniprogram/pages/bp-drafts`

核心接口：

- `GET /api/user-data/bp-drafts`
- `POST /api/user-data/bp-drafts`
- `DELETE /api/user-data/bp-drafts/{draftKey}`
- `POST /api/bp/export/{format}`，其中 `format` 支持 `doc` 或 `pdf`。

### 6. 行业图谱

行业图谱用于呈现行业结构、产业链关系、机会点和创业切入方向。当前该模块以前端内置数据为主，适合作为行业调研和项目路演材料准备的辅助入口。

适用场景：

- 创业方向调研。
- 行业竞品分析。
- 商业计划书市场分析。
- 寻找产业链上下游机会。

对应页面：

- `miniprogram/pages/industry-atlas`

### 7. 图说解读

图说解读模块面向图片、图表、报告截图等复杂信息场景，目标是帮助用户更快理解图像中的关键信息。当前小程序已具备页面结构，后续可以继续升级为多模态 AI 分析能力。

对应页面：

- `miniprogram/pages/image-interpretation`

### 8. 资源对接

资源对接页面当前主要展示校园机会、技能服务、灵活服务和基础便利类资源，支持搜索和分类筛选。它可以作为后续创业服务资源库、场地预约、活动报名和服务申请的基础版本。

对应页面：

- `miniprogram/pages/resource`

### 9. 专家对接

专家对接模块用于连接创业导师、行业专家、财务顾问、法律顾问等外部专业资源。当前以前端内置专家列表和详情弹窗为主，后续可扩展为真实专家库、预约流程和咨询记录。

对应页面：

- `miniprogram/pages/expert-connect`

### 10. 我的与个人工作台

“我的”页面是用户个人工作台，深度应用极简排版与黑白对比的高级 UI 设计，聚合登录状态、个人资料、数据统计、常用服务、收藏、记录、草稿、通知和设置。

主要能力：

- 登录、注册、退出。
- 查看和编辑个人资料。
- 查看政策收藏、职业记录、BP 草稿、通知数量等统计。
- 进入设置、通知、收藏、草稿和记录页面。

对应页面：

- `miniprogram/pages/mine`
- `miniprogram/pages/account-profile`
- `miniprogram/pages/settings`
- `miniprogram/pages/notifications`

核心接口：

- `POST /api/auth/login`
- `POST /api/auth/register`
- `POST /api/auth/logout`
- `GET /api/auth/userInfo`
- `PUT /api/auth/userInfo`
- `GET /api/user-data/stats`
- `GET /api/user-data/notifications`
- `POST /api/user-data/notifications`
- `PUT /api/user-data/notifications/{id}/read`
- `PUT /api/user-data/notifications/read-all`

## 六、后端服务模块

### 1. 基础服务

基础健康检查接口用于确认后端是否正常启动：

- `GET /`
- `GET /health`
- `GET /api/test`
- `POST /api/test`

### 2. 认证服务

`LoginController` 提供账号密码登录、注册、退出和用户信息维护能力。当前已具备本地账号体系，后续可以继续扩展微信登录并与现有用户体系打通。

接口：

- `POST /api/auth/login`
- `POST /api/auth/register`
- `POST /api/auth/logout`
- `GET /api/auth/userInfo`
- `PUT /api/auth/userInfo`

### 3. AI 服务

`AiAssistantController` 与 `DeepSeekService` 负责 AI 会话、聊天、历史记录和会话查询。`application.yml` 中通过 `DEEPSEEK_API_KEY` 注入密钥，默认模型为 `deepseek-chat`。

配置示例：

```yaml
deepseek:
  api-key: ${DEEPSEEK_API_KEY:}
  api-url: https://api.deepseek.com/v1/chat/completions
  model: deepseek-chat
```

### 4. 政策服务

`PolicyDataController` 与 `PolicyDataService` 提供政策列表、详情和政策问答能力。政策列表支持 `keyword` 查询，可配合导入脚本持续扩充政策数据。

### 5. 职业匹配服务

`OccupationController` 提供专业、技能、性格选项和匹配计算接口。业务逻辑位于 `OccupationService`，实体数据由 `OccupationMapper` 访问。

### 6. 用户数据服务

`UserDataController` 提供个人数据沉淀能力，包括：

- 用户统计。
- BP 草稿。
- 职业匹配记录。
- 政策收藏。
- 通知消息。

这些接口依赖登录后的 Authorization Token，用于解析当前用户身份。

### 7. BP 导出服务

`BpExportController` 支持将 BP 内容导出为 `doc` 或 `pdf`。当前实现偏轻量：Word 使用 HTML 文档输出，PDF 使用后端绘制页面并组装 PDF 文件，适合作为原型阶段的导出能力。

### 8. Banner 服务

`BannerController` 负责首页轮播图展示和管理能力：

- `GET /api/banner/list`
- `GET /api/banner/admin/list`
- `POST /api/banner/admin/upload`
- `PUT /api/banner/admin/update`
- `DELETE /api/banner/admin/delete/{id}`

### 9. 通用配置

- `CorsConfig`：支持跨域请求，便于本地调试。
- `WebConfig`：将 `/uploads/**` 映射到项目根目录下的 `uploads`。
- `MybatisPlusConfig`：MyBatis Plus 配置。
- `GlobalExceptionHandler`：统一异常处理。
- `Result` / `ResultCode`：统一响应结构。

## 七、数据与脚本

### 1. 数据库脚本

- `creative_engine.sql`：项目主要数据库初始化脚本。
- `scripts/user_personal_data_schema.sql`：用户收藏、草稿、记录、通知等个人数据表。
- `backend/sql/policy_item_schema.sql`：政策数据相关表结构。
- `backend/sql/policy_item_seed.sql`：政策种子数据。
- `backend/sql/policy_item_search.sql`：政策查询示例。

### 2. 政策抓取脚本

- `scripts/import-gov-graduate-policies.js`：按关键词抓取公开政策并生成 SQL。
- `scripts/schedule-gov-policy-import.js`：定时执行政策导入。
- `scripts/generated/graduate_policies.sql`：生成的政策 SQL。

使用示例：

```bash
node scripts/import-gov-graduate-policies.js "--keyword=毕业生 创业,大学生 就业" --pages=1 --output=scripts/generated/graduate_policies.sql
mysql -uroot -p creative_engine < scripts/generated/graduate_policies.sql
```

## 八、运行方式

### 1. 后端运行

环境要求：

- JDK 17+
- Maven 3.6+
- MySQL 8.0+

初始化数据库：

```sql
CREATE DATABASE creative_engine DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

导入脚本：

```bash
mysql -uroot -p creative_engine < creative_engine.sql
mysql -uroot -p creative_engine < scripts/user_personal_data_schema.sql
```

配置敏感信息：

```bash
DB_PASSWORD=your_mysql_password
DEEPSEEK_API_KEY=your_deepseek_api_key
```

启动服务：

```bash
mvn spring-boot:run
```

默认访问地址：

```text
http://localhost:8080
```

### 2. 小程序运行

1. 打开微信开发者工具。
2. 导入 `miniprogram` 目录。
3. 配置 AppID，或使用测试号进行本地调试。
4. 本地调试时勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
5. 确认 `miniprogram/app.js` 中接口地址为：

```js
apiBase: 'http://localhost:8080/api'
```

## 十、项目特色

### 1. 围绕真实创业链路设计

项目不是简单堆叠页面，而是围绕早期创业用户的连续路径展开：先了解自己适合做什么，再查询政策和行业机会，再通过 AI 辅助整理材料，最后把收藏、记录和草稿沉淀到个人工作台。

### 2. 移动端轻量入口

微信小程序降低了使用门槛，适合校园创业活动、创新创业比赛、孵化器服务、政策宣讲和日常咨询场景。

### 3. 前后端结构清晰

小程序负责交互和展示，后端负责数据、AI 和业务接口。模块边界清晰，便于后续扩展管理后台、Web 端或更多数据服务。

### 4. AI 能力可持续扩展

当前已经接入 DeepSeek 对话能力，后续可以继续扩展到政策精准推荐、BP 深度生成、图像理解、项目诊断报告、路演问答模拟等场景。

### 5. 具备个人数据沉淀能力

项目已经从纯展示页面升级到可保存、可查看、可继续编辑的工作台形态，包括政策收藏、职业记录、BP 草稿和通知消息。

## 十二、总结

创享引擎是一个以青年创业者为中心的智能创业服务平台。它通过微信小程序提供轻量、便捷的使用入口，通过 Spring Boot 后端提供稳定的数据和 AI 服务，通过政策、职业、行业、资源、BP 和个人工作台模块覆盖用户从想法到行动的关键步骤。

当前项目已经具备较完整的业务雏形和技术结构，适合作为课程设计、毕业设计、创新创业项目原型或真实创业服务平台的基础版本。后续如果继续围绕数据沉淀、AI 深度赋能、真实资源对接和管理后台建设进行增强，可以逐步发展为面向校园创业、青年创业和地方创新服务的综合平台。
